# v7.0 문서 검토 및 이슈 생성 가이드

v7.0 계획 문서들을 효과적으로 검토하고 GitHub 이슈로 전환하기 위한 추천 순서입니다.

---

## 📅 1단계: 검토 순서 (하향식)

전체적인 방향성을 먼저 잡고, 세부 구현 내용으로 들어가는 순서를 추천합니다.

### 1. 🌟 `v7.0_roadmap.md` (로드맵)
- **목적**: v7.0의 전체 비전("자율 에이전트")과 3단계(Phase) 구성을 이해합니다.
- **체크 포인트**:
  - 제안된 3개의 Phase가 비즈니스 및 기술적 목표에 부합하는지 확인
  - Phase별 일정(마일스톤)이 현실적인지 검토

### 2. 📋 `v7.0_task_list.md` (이슈 리스트)
- **목적**: 로드맵의 각 Phase가 어떤 구체적인 작업(Issue)으로 쪼개지는지 확인합니다.
- **체크 포인트**:
  - 누락된 기능이나 작업이 없는지 확인
  - 각 이슈의 범위가 적절한지 (너무 크거나 작지 않은지)

### 3. 🏗️ `v7.0_phase1_design.md` (Phase 1 설계)
- **목적**: 당장 시작할 Phase 1(LangGraph Agent)의 기술적 설계를 검토합니다.
- **체크 포인트**:
  - `LangGraph` 도입의 타당성
  - `입력 → 분석 → 검색 → 분류` 흐름이 적절한지

### 4. 🛠 `v7.0_phase1_setup.md` (Phase 1 구현 가이드)
- **목적**: 실제 코딩을 위한 준비 사항을 확인합니다.
- **체크 포인트**:
  - 필요한 라이브러리(`langgraph` 등) 설치 가능 여부
  - 초기 폴더 구조 변경(`backend/agent/`)에 대한 이해
