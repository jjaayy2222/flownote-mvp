# FlowNote v7.0 로드맵: 자율 지식 에이전트

## 🌟 비전
**"수동적 분류기에서 능동적 지식 에이전트로"**

FlowNote v7.0은 단순히 업로드된 파일을 분류하는 도구에서, 사용자 맥락을 이해하고 지식을 능동적으로 관리하며 자연어로 상호작용하는 **자율 에이전트**로 진화하는 것을 목표로 합니다.

---

## 📅 Phase 개요

### Phase 1: 고급 AI 에이전트 (LangGraph)
> **목표**: 핵심 분류 엔진을 상태 기반 추론 에이전트로 업그레이드

- **목적**: 단순 LLM 호출을 인지 아키텍처(LangGraph)로 대체
- **주요 기능**:
  - **다단계 추론**: `분석` → `맥락 참조` → `분류` → `검증`
  - **자가 수정**: 낮은 신뢰도 분류를 자동으로 재평가
  - **맥락 인식 메모리**: Redis를 사용하여 과거 사용자 피드백 및 프로젝트 맥락 기억

### Phase 2: 지능형 자동화 (RAG & 채팅)
> **목표**: "문서와 대화하기" 및 의미론적 이해 구현

- **목적**: 대화형 지식 검색을 위한 RAG(검색 증강 생성) 구현
- **주요 기능**:
  - **채팅 인터페이스**: 보관된 문서에 대해 질문 (예: "Q1 보고서 요약해줘")
  - **의미론적 검색**: 키워드가 아닌 의미로 문서 찾기 (벡터 데이터베이스)
  - **스마트 링킹**: 다른 PARA 카테고리의 관련 문서 자동 제안

### Phase 3: 멀티 플랫폼 통합
> **목표**: 에이전트의 활동 범위를 웹 대시보드 밖으로 확장

- **목적**: 원활한 워크플로우를 위한 커뮤니케이션 도구 통합
- **주요 기능**:
  - **이메일 통합**: FlowNote로 이메일을 전달하여 자동 분류
  - **Slack/Discord 봇**: 채팅 앱을 통해 일일 요약 수신 및 문서 조회
  - **모바일 접근**: 반응형 PWA 또는 네이티브 앱 래퍼

## 🛠 기술 스택 업데이트

| 구성 요소 | 현재 (v6.0) | v7.0 업그레이드 |
|-----------|-------------|----------------|
| **AI 오케스트레이션** | LangChain (기본 체인) | **LangGraph** (상태 기반 에이전트) |
| **메모리** | 무상태 | **Redis** (대화 버퍼) |
| **검색** | FAISS (기본) | **ChromaDB / Pinecone** (하이브리드 검색) |
| **프론트엔드** | Next.js (대시보드) | Next.js + **AI SDK** (스트리밍 채팅) |
| **백엔드** | FastAPI (REST/WS) | FastAPI + **비동기 에이전트** |

## 📝 마일스톤 전략

1. **기초 작업 (1-2주차)**: LangGraph 설정, State 정의, 메모리 통합
2. **핵심 에이전트 (3-4주차)**: 추론 루프 구현, 자가 수정 로직
3. **RAG 시스템 (5-6주차)**: 임베딩 파이프라인, 벡터 DB 최적화, 채팅 UI
4. **통합 (7-8주차)**: 외부 API (Slack/Email), 엔드투엔드 테스팅
