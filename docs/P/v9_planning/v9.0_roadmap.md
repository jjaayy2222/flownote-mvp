# v9.0 Roadmap: Adaptive Intelligence & Personalization

> 상위 이슈: TBD | 마일스톤: v9.0 Release | 기준 브랜치: develop (v8.0)

## 🎯 v9.0 비전

v8.0에서 구축한 **데이터 플라이휠(Data Flywheel)** 위에,  
v9.0에서 **스스로 개선되는 적응형 AI 코어(Adaptive Intelligence Core)** 구현하기.

```
v8.0: 데이터 수집 → 평가 → Fallback (인프라)
v9.0: 파인튜닝 → 개인화 → 사용자 적응 (인텔리전스)
```

## 📋 Phase 1: Golden Dataset 기반 파인튜닝 자동화
> 브랜치: `feature/issue-1045-adaptive-finetune`  
> 이슈 번호: [#1045] | 마일스톤: `v9.0 - Adaptive Intelligence`

### 배경
- v8.0에서 `golden_dataset_service.py`가 파인튜닝용 `.jsonl` 파일을 자동 수집.  
- 이제 이 데이터를 실제 LLM 파인튜닝 파이프라인에 연결하기.

### 세부 Tasks
- [ ] OpenAI Fine-tuning API 또는 LoRA(PEFT) 파인튜닝 스크립트 구현
- [ ] 파인튜닝 완료 모델을 LangGraph 에이전트에 자동으로 교체하는 핫스왑(Hot-swap) 메커니즘
- [ ] 파인튜닝 성공/실패 결과를 Redis + Discord 알림으로 연동
- [ ] 파인튜닝 이전/이후 성능 비교 지표(BLEU, 사용자 평점 변화) 자동 산출

### 이슈 메타정보
- **Labels**: `enhancement`, `ai/finetune`, `backend`
- **Milestone**: `v9.0`

## 📋 Phase 2: 사용자 개인화 RAG (Personalized RAG)
> 브랜치: `feature/issue-1046-personalized-rag`  
> 이슈 번호: [#1046] | 마일스톤: `v9.0 - Adaptive Intelligence`

### 배경
- 현재 RAG는 모든 사용자에게 동일한 전역 FAISS 인덱스 사용.  
- 사용자별 상호작용 패턴을 분석해 개인화된 컨텍스트 창을 제공하기.

### 세부 Tasks
- [ ] 사용자별 임베딩 서브-인덱스 생성 및 관리:
  - **Path Strategy**: `${STORAGE_BASE_PATH}/user_data/idx/{id}.faiss` (멀티 리전/환경 대응을 위해 베이스 경로 환경 변수화)
- [ ] 사용자 검색 히스토리 기반 관심 토픽 클러스터링 (k-means)
- [ ] 개인화 컨텍스트 가중치 라우팅: 개인 인덱스(60%) + 전역 인덱스(40%) 혼합
- [ ] GDPR 준수 삭제 정책 수립 및 기술적 구현:
  - **Vector Data**: 실제 파일 및 메모리 인덱스 영구 삭제 (Hard Delete)
  - **Interaction Logs**: PII(Email, IP 등) 전면 삭제 및 고유 식별자의 **Salted Hash** 전환으로 재식별 리스크 원천 차단
  - **Audit Logging**: 삭제 요청 및 처리 결과에 대한 감사 로그 자동 생성 (보관: 90일)

### 이슈 메타정보
- **Labels**: `enhancement`, `ai/rag`, `privacy`, `backend`
- **Milestone**: `v9.0`

## 📋 Phase 3: 실시간 스트리밍 & 프론트엔드 UX
> 브랜치: `feature/issue-1047-realtime-streaming`  
> 이슈 번호: [#1047] | 마일스톤: `v9.0 - Adaptive Intelligence`

### 배경
- 현재 응답은 전체 생성 완료 후 반환됨.  
- 스트리밍 응답으로 사용자 체감 응답 속도를 극적으로 개선.

### 세부 Tasks
- [ ] LangGraph 에이전트에 LangChain `StreamingCallbackHandler` 적용

- [ ] Next.js 프론트엔드에서 스트리밍 응답 청크 수신 및 실시간 렌더링
- [ ] 타이핑 인디케이터, 마크다운 실시간 렌더링, 코드 블록 하이라이팅

## 📋 Phase 4: 노트 기반 지식 그래프 (Knowledge Graph)
> 브랜치: `feature/issue-1048-knowledge-graph`  
> 이슈 번호: [#1048] | 마일스톤: `v9.0 - Adaptive Intelligence`

### 배경
- Obsidian 기반 노트들을 플랫 벡터 인덱스뿐 아니라 **관계형 지식 그래프**로도 표현해 더 풍부한 컨텍스트 추론을 가능하게 하기.

### 세부 Tasks
- [ ] `networkx` 또는 `neo4j`를 활용한 노드(노트) + 엣지(연결) 그래프 구성
- [ ] LangGraph 에이전트가 그래프 탐색을 RAG와 병행하는 `GraphRAG` 라우터 구현
- [ ] 프론트엔드 그래프 시각화: `react-force-graph` 기반 지식 맵 뷰
- [ ] 연결되지 않은 고립 노트 자동 감지 및 관련 노트 추천 기능

### 이슈 메타정보
- **Labels**: `enhancement`, `ai/graph`, `frontend`, `backend`
- **Milestone**: `v9.0`
