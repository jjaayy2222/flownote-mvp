(🚨 중요: 각 카테고리의 특성을 먼저 파악하고 분류하세요)

## 🎯 카테고리 정의 (Context Cue 포함)

### 1. Projects (P) - 명확한 기한과 목표가 있는 작업
**핵심 신호:**
- ⏰ 시간 표현: "by", "until", "~까지", "기한", "마감", "deadline", "due", "기간"
- 🎯 목표 동사: "완성", "달성", "구축", "론칭", "제출", "발표", "구현"
- 📍 진행 상태: "진행 중", "계획", "시작", "진행하기", "한다", "구현", "배포"

**예시:**
- "Q4 마케팅 캠페인 11월 30일까지 론칭"
- "프로젝트 제안서 마감일 10월 31일"
- "새로운 기능 개발 중, 12월 완료 예정"

---

### 2. Areas (A) - 지속적으로 유지하는 관심 영역
**핵심 신호:**
- 🔄 지속성 표현: "ongoing", "반복적", "지속적", "관리", "유지", "개선", "항상", "계속"
- 💼 책임 영역: "역할", "분야", "영역", "책임", "담당", "전담"
- ⚡ 특징: 구체적 마감 기한이 없음

**예시:**
- "마케팅 전략 및 브랜드 개선 (지속적 유지)"
- "직원 관리 및 팀 성과 평가"
- "기술 스택 학습 및 개발 역량 강화"
- "건강 유지 및 운동 습관"

---

### 3. Resources (R) - 참고용 자료, 정보, 가이드
**핵심 신호:**
- 📚 참조 키워드: "guide", "reference", "manual", "설명서", "자료", "가이드", "정보", "모음", "모아놓은"
- 🔍 정보성 문체: "~하는 방법", "~란?", "설명", "정의", "기술 문서", "튜토리얼"
- 🎓 학습/도구: "튜토리얼", "템플릿", "문서", "링크", "API 문서", "정리", "정보"

**예시:**
- "Python 코딩 튜토리얼 및 예제 모음"
- "AWS API 문서"
- "마케팅 전략 템플릿"
- "Git 사용법 가이드"

---

### 4. Archives (AR) - 완료되었거나 더 이상 필요 없는 항목
**핵심 신호:**
- ✅ 완료 표현: "completed", "finished", "done", "완료", "종료", "완료됨", "끝남"
- 📅 과거형 표기: "지난 ~", "2024년", "구식", "미사용", "보관", "deprecated", "옛날"
- 🚫 비활성: "폐지", "폐기", "더 이상", "사용하지 않음", "만료"

**예시:**
- "2024년 마케팅 캠페인 - 완료됨"
- "구식 API 문서 - 더 이상 지원 안 함"
- "이전 프로젝트 아카이브"

## 🎯 분류 의사 결정 프로세스

**Step 1: 시간 표현 체크**
- "기한", "~까지", "deadline", "due", "마감" 있음? → **Projects 가능성 높음**
- "지속적", "반복적", "ongoing", "항상" 있음? → **Areas 가능성 높음**

**Step 2: 콘텐츠 특성 체크**
- "참고", "가이드", "설명서", "tutorial" 있음? → **Resources**
- "완료", "종료", "보관", "archived", "완료됨" 있음? → **Archives**

**Step 3: 행동 동사 체크**
- Action verb (만들다, 달성하다, 구축하다, 론칭하다)? → **Projects**
- Maintenance verb (관리하다, 유지하다, 개선하다, 학습하다)? → **Areas**
- Descriptive (설명하다, 정의하다, 제공하다)? → **Resources**

**Step 4: 모호한 경우 우선순위**
```
⚠️ 여러 특징이 겹칠 경우 우선순위:

Projects > Areas > Resources > Archives

예시1: "마케팅 전략 가이드"
→ "가이드"는 Resource BUT "마케팅 전략"은 Area
→ "11월까지" 같은 기한 있나? 없으면 Resource

예시2: "2024년 마케팅 전략 보관"
→ "보관", "2024년" 있음 → Archives
```

```

  "category": "Projects|Areas|Resources|Archives",
  "confidence": 0.0-1.0,
  "reasoning": "선택 이유 (2-3줄, 한국어)",
  "detected_cues": ["감지된_키워드1", "감지된_키워드2", "감지된_키워드3"]
}
```

**reasoning 예시:**
- Projects: "마감일(11월까지) + 구체적 목표(론칭)로 Projects 분류"
- Areas: "지속적 + 책임 영역(마케팅) 관리 → Areas 분류"
- Resources: "참고 자료 + 가이드 성격 → Resources 분류"
- Archives: "완료 표현 + 연도 표기(2024년) → Archives 분류"
