## 🎯 카테고리 정의 (Context Cue 포함)

### 1. Projects (P) - 명확한 기한과 목표가 있는 작업
**핵심 신호:**
- ⏰ 시간 표현: "by", "until", "~까지", "기한", "마감", "deadline", "due", "기간"
- 🎯 목표 동사: "완성", "달성", "구축", "론칭", "제출", "발표", "구현", "개발"
- 📍 진행 상태: "진행 중", "계획", "시작", "진행하기", "한다", "구현", "배포"

**예시:**
- "Q4 마케팅 캠페인 11월 30일까지 론칭"
- "프로젝트 제안서 마감일 10월 31일"
- "새로운 기능 개발 중, 12월 완료 예정"

---

### 2. Areas (A) - 지속적으로 유지하는 관심 영역
**핵심 신호:**
- 🔄 지속성 표현: "ongoing", "반복적", "지속적", "관리", "유지", "개선", "항상", "계속"
- 💼 책임 영역: "역할", "분야", "영역", "책임", "담당", "전담"
- ⚡ 특징: 구체적 마감 기한이 없음

**예시:**
- "마케팅 전략 및 브랜드 개선 (지속적 유지)"
- "직원 관리 및 팀 성과 평가"
- "기술 스택 학습 및 개발 역량 강화"
- "건강 유지 및 운동 습관"

---

### 3. Resources (R) - 참고용 자료, 정보, 가이드
**핵심 신호:**
- 📚 참조 키워드: "guide", "reference", "manual", "설명서", "자료", "가이드", "정보", "모음"
- 🔍 정보성 문체: "~하는 방법", "~란?", "설명", "정의", "기술 문서", "튜토리얼", "문서"
- 🎓 학습/도구: "튜토리얼", "템플릿", "링크", "API 문서", "정리", "정보 모음"

**예시:**
- "Python 프로그래밍 가이드 및 튜토리얼 모음"
- "AWS 클라우드 설명서 및 모범 사례"
- "마케팅 템플릿 및 도구 모음"

---

### 4. Archives (Ar) - 완료되었거나 더 이상 필요 없는 항목
**핵심 신호:**
- ✅ 완료 표현: "완료", "완성", "끝남", "종료", "폐기", "더 이상 사용 안 함"
- 📦 보관 상태: "보관", "archive", "과거", "이전", "레거시", "폐지됨"
- ⏳ 시간 표현: "작년", "지난해", "과거", "이전 버전"

**예시:**
- "2024년 마케팅 캠페인 (완료됨)"
- "레거시 시스템 마이그레이션 완료"
- "지난해 분기별 보고서"

---

## 📋 입력 데이터 구조

다음과 같은 텍스트가 주어집니다:
- 제목 (title)
- 본문 내용 (content)
- 필요시 메타데이터 (metadata)

---

## 📊 출력 포맷 (JSON)

**반드시 다음과 같은 JSON 형식으로 응답하세요:**

```
{
  "category": "Projects|Areas|Resources|Archives",
  "confidence": 0.0 - 1.0,
  "reasoning": "분류 이유 설명",
  "signals_detected": {
    "temporal_cues": ["시간 관련 단어들"],
    "action_verbs": ["동작 동사들"],
    "keywords": ["핵심 키워드들"]
  }
}
```

**필드 설명:**
- `category`: Projects, Areas, Resources, Archives 중 하나
- `confidence`: 0.5 이상 권장 (낮으면 Resources로 기본값)
- `reasoning`: 왜 이 카테고리를 선택했는가
- `signals_detected`: 감지된 신호들 (문제 추적용)

---

## 🔍 분류 알고리즘

**Step 1: 시간 신호 확인**
- 명확한 마감일/기한이 있는가? → Projects
- 지속적 표현이 있는가? → Areas
- 참조/안내 표현이 있는가? → Resources
- 완료 표현이 있는가? → Archives

**Step 2: 내용 심화 분석**
- 액션 동사 분석 (동작 지향적? 유지 지향적? 참고 지향적?)
- 키워드 분포 분석
- 문맥 일관성 확인

**Step 3: Confidence 계산**
- 신호가 명확할수록 높게
- 신호가 모호하면 0.5 ~ 0.7
- 신호가 충돌하면 0.3 ~ 0.5

**Step 4: Fallback 규칙**
- Confidence < 0.5 → Resources (기본값)
- 모호한 경우 → Areas (지속적 관심 영역)
